# 乗算式 vs 加算式ダメージの比較

## 概要
多くのゲームのダメージ計算は「加算式」か「乗算式」、またはその組み合わせで構成される。両者の違いとビルド最適化への影響を整理した。

## 加算式（Additive）

```
Damage = Base × (1 + Bonus_A + Bonus_B + Bonus_C)
```

- すべてのボーナスが**同じプールに合算**される
- ボーナスが増えるほど、1つあたりの上昇幅が**相対的に小さくなる**
- 例: 50%持ちに50%追加 → 100%/150% = 1.67倍増加率、100%持ちに50%追加 → 150%/200% = 1.25倍増加率

## 乗算式（Multiplicative）

```
Damage = Base × (1 + A) × (1 + B) × (1 + C)
```

- 各項が**独立して掛け合わされる**
- 項が異なるカテゴリなら、どの項を伸ばしても一定の効率が得られる
- **バランスよく各項を伸ばすのが最適**（AM-GM不等式）

## 実例

| ゲーム | 方式 | 備考 |
|---|---|---|
| Arknights: Endfield | 乗算 | 各倍率項が独立 |
| Path of Exile | 加算+乗算の混合 | カテゴリ内加算、カテゴリ間乗算 |
| Genshin Impact | 乗算ベース | ATK/DMG Bonus/Crit が独立 |

## 考察
乗算式のゲームでは、**1つの項に集中投資するよりも、低い項を底上げする方が総合効率が高い**。ビルド最適化の基本原理。
